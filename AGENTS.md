# Temu Pro Extension — AI Agents Guidelines (AGENTS.md)

Цей файл містить ключові інструкції та найкращі практики для AI-агентів (зокрема моделей у фреймворку Antigravity), які працюють над підтримкою та розширенням проєкту "Temu Smart Score & Bar (UA) — Pro".

## Основна парадигма співпраці
Агенти повинні діяти як автономні інженери-напарники (Pair Programmers), які завжди віддають пріоритет:
1. **Збереженню існуючої архітектури**: Ніколи не переписуйте працюючий код без явної необхідності або запиту. Притримуйтесь існуючих шаблонів (Vanilla JS, `chrome.storage.local`, MV3 Service Workers).
2. **Точності та мінімалізму**: Зміни мають бути атомарними. Використовуйте `replace_file_content` для точкових модифікацій, замість того, щоб переписувати весь файл.
3. **Локалізації**: Всі інтерфейсні тексти, коментарі в коді та документація (крім назв змінних) мають бути українською мовою.

## Робота з Antigravity Framework
Framework Antigravity надає розширені можливості навігації по коду та виконання команд.
- **Пошук рішень**: Завжди починайте виконання складних завдань з `grep_search` або `view_file` по існуючим файлам, щоб зрозуміти контекст (наприклад, де знаходиться потрібний DOM-елемент або як працює IPC повідомлення).
- **Спілкування з користувачем**: Запитуйте дозволи або уточнення через `notify_user` тільки тоді, коли зустрічаєтесь з критичними неясностями архітектури або блокерами. Уникайте постійного відволікання користувача на дрібниці.
- **Тестування**: Використовуйте Puppeteer-скрипти у папці `tests/` через `run_command` для візуального підтвердження та налагодження розширення без ручного втручання.
- **Артефакти**: Документуйте виконану роботу у внутрішніх артефактах (наприклад, `walkthrough.md`, `task.md`), щоб зберігати цілісний контекст для себе та майбутніх сесій.

## Специфіка проєкту Temu Pro
- **DOM маніпуляції**: Динамічні сайти типу Temu часто перемальовують DOM (через React/Vue). Наші скрипти (`content.js`) покладаються на `MutationObserver` або періодичні виклики `setInterval` (`processCards`). Завжди перевіряйте стійкість ваших CSS-селекторів.
- **Події та IPC (Inter-Process Communication)**: Через обмеження MV3, `content.js` в ізольованому світі спілкується з `sidepanel.js` (UI) та `background.js` через `chrome.runtime.sendMessage` та `chrome.storage.onChanged`. Уникайте збережень великих обсягів даних занадто часто (використовуйте debounce або прямі IPC для real-time реакцій).
- **Стилізація**: Надавайте перевагу CSS-змінним та Flexbox. Всі класи повинні мати префікс `.tpw-` (Temu Pro Window) або `.st-` (Settings UI), щоб не конфліктувати зі стилями цільового сайту.

## Правило "Завжди перевіряй"
Якщо ви оновлюєте логіку калькуляції `Smart Score` або змінюєте алгоритм розрахунку балів, переконайтесь, що:
1. Оновлено tooltip у `content.js`.
2. Оновлено логіку в `processCards`.
3. Оновлено повзунки та відображення налаштувань у `sidepanel.html` та `sidepanel.js`.
4. Загальна сума балів дорівнює 100 (80 основа + 20 бонуси).
