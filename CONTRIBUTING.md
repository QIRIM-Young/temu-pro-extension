# CONTRIBUTING.md

Дякуємо за ваш інтерес до розвитку проєкту **Temu Smart Score & Bar (UA) — Pro**! 🚀
Будь-які покращення, виправлення помилок (bug fixes) та нові ідеї високо вітаються. Будь ласка, ознайомтеся з наведеними нижче стандартами для забезпечення стабільності та чистоти кодової бази.

## Основні Правила Кодової Бази

1.  **Manifest V3 Only:** Проєкт суворо дотримується стандартів Manifest V3 (MV3). Не використовуйте застарілі Chrome API (наприклад, `background pages`). Для фонових задач використовуйте виключно Service Workers (`background.js`).
2.  **Без Фреймворків (Vanilla First):** В основі розширення лежить чистий Vanilla JavaScript. Ми відмовляємось від важких фреймворків типу React, Vue, чи jQuery заради швидкості, легкої ваги та безпечного проходження рев'ю для Chrome Web Store.
3.  **Українська Локалізація:** Усі тексти користувацького інтерфейсу (UI), сповіщення, console.logs, а також документація та коментарі в коді повинні бути написані **українською мовою**. Імена змінних та функцій — класично англійською формату `camelCase`.
4.  **Ізоляція CSS:** Щоб запобігти зламів верстки цільового сайту (Temu.com), усі класи, що ін'єктуються безпосередньо у DOM сайту, повинні мати префікс виду `temu-` або `tpw-` (Temu Pro Window).
5.  **Безпека CSP:** Заборонено використання `eval()`, `setTimeout(string)`, або включення сторонніх неперевірених CDN-скриптів безпосередньо в код розширення (MV3 Security constraints).

## Розуміння Архітектури

Ми розділили логіку розширення для максимальної ізольованості та зручності підтримки:
*   **`background.js` (Service Worker):** Життєвий цикл фонового скрипта короткий (зупиняється в стані простою). Застосовуйте `chrome.storage` для персистентності. Цей скрипт є ідеальним місцем для CORS API запитів (наприклад: ExchangeRate-API).
*   **`content.js` (Content Script):** Завантажується безпосередньо у контекст сторінки Temu. Відповідає **виключно** за:
    1. Парсинг цін, рейтингів і карток товарів.
    2. Розміщення математично обрахованих бейджів (`tooltips`) на товарах.
    3. Створення "головного вікна" (`#temu-pro-window`), всередину якого вбудовується ізольований `<iframe src=".../sidepanel.html">`.
*   **`sidepanel.html` та `sidepanel.js` (UI Iframe):** Відображає панель налаштувань (схована за замовчуванням у фіксованому вікні поверх сайту Temu або у нативній Sidebar панелі Chrome). Усі налаштування алгоритму Smart Score, повзунки та перемикачі знаходяться тут. 

### IPC: Спілкування між вікнами
Оскільки `sidepanel` інтегрований у `content.js` як Iframe, вони використовують нативний стандарт обміну повідомленнями `window.postMessage()` (або `chrome.runtime.sendMessage` для звя'зку з Service Worker). Якщо вам необхідно передати подію з сайту Temu (hover, click) в панель, створюйте відповідний `postMessage` Event Listener.

## 🛠 Процес додавання змін (Pull Request)

1.  **Форк (Fork):** Зробіть форк репозиторію.
2.  **Гілка (Branch):** Створіть нову гілку для вашої фічі або багфіксу (`git checkout -b feature/amazing-new-feature` або `git checkout -b fix/annoying-bug`).
3.  **Додайте код:** Напишіть свій геніальний код дотримуючись стандартів проєкту.
4.  **Тестування:** Завантажте папку `extension/` як "Unpacked Extension" в Chrome (`chrome://extensions/`) та перевірте на реальному сайті Temu. (Або скористайтеся автоматизованими тестами в папці `/tests`).
5.  **Синтаксис та Помилки:** Відкрийте вкладку помилок (Errors) у `chrome://extensions`, щоб переконатись у відсутності Runtime Error / CSP Warning фіксацій.
6.  **Коміт:** Зробіть коміт за допомогою зрозумілих повідомлень, бажано у форматі Conventional Commits (Наприклад: `feat: додано нову систему кешування для курсу валют`).
7.  **Push & PR:** Запуште код у ваш форк та відкрийте Pull Request в оригінальному репозиторії.
